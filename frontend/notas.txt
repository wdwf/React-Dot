WEBPACK
  Webpack vai gerenciar a leitura de arquivos CSS, JavaScript e assets
  de seu projeto, empacotando esse conteúdo para desenvolvimento ou produção.
  Ele irá funcionar de forma recursiva, um arquivo funciona como ponto de partida,
  a partir desse ponto inicial ele irá varrer todos os arquivos que estão 
  conectados em sua aplicação e para cada caso teremos um tratamento específico.

  fazendo assim um module bundler (um empacotador de módulos para aplicaçoes JS)
  No geral um bundles é um arquivo final onde o webpack 
  pega todos os modulos e vai empacotar dentro de um unico arquivo,
  ele trabalha com grafos então ele consegue criar toda uma arvore

  A vantagem de utilizar esse recurso é que evitaremos carregar diversos arquivos JS com
  diferentes tags amenizando problemas de performace na aplicação 

  conceitos:
  o webpack precisará do arquivo de 
  *Entry: que é um ponto de entrada para buscar
  todos os modulos e dependencias.
  *Output: É para determinar quais são os bundler que o webpack ira emitir
  *Loaders:
  *Plugins:
